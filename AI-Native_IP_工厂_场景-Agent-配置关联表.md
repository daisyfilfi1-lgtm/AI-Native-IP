## 场景一：🎯 推荐选题

### 使用场景
IP每天早上打开工作台，看到系统推荐的3个选题卡片，每个卡片显示评分、预估播放、匹配理由。

### 调用Agent流程

```
┌────────────────────────────────────────────────────────────────┐
│                     推荐选题调用链                              │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  ① Strategy Agent (核心)                                       │
│     ├─ 输入：IP画像 + 竞品监控清单 + 平台热点数据               │
│     ├─ 处理：                                                  │
│     │   • 抓取竞品低粉爆款 (<10万粉，>1万赞)                   │
│     │   • 四维评分算法计算每个选题得分                          │
│     │   • 预测播放量 (基于历史数据 + 话题热度)                  │
│     └─ 输出：TOP 10候选选题                                     │
│                    ↓                                           │
│  ② Memory Agent (辅助)                                         │
│     ├─ 输入：候选选题关键词                                     │
│     ├─ 处理：语义检索IP素材库，匹配相关故事                     │
│     └─ 输出：每个选题的匹配理由                                 │
│         例："基于你的故事_003匹配"                              │
│                    ↓                                           │
│  ③ Analytics Agent (辅助)                                      │
│     ├─ 输入：选题历史表现数据                                   │
│     ├─ 处理：预测性模型计算预估播放量                           │
│     └─ 输出："50-80万"、"预估完播率32%"                        │
│                    ↓                                           │
│  前端展示：3个选题卡片                                           │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 关键配置项

| 配置项 | 所在Agent | 配置路径 | 对场景的影响 |
|--------|-----------|----------|--------------|
| **四维评分权重** | Strategy Agent | 策略Agent → 评分规则 | 决定选题排序优先级<br>• 流量权重高 → 推荐热门话题<br>• 变现权重高 → 推荐产品相关内容<br>• IP契合度权重高 → 推荐符合人设的话题 |
| **竞品监控清单** | Strategy Agent | 策略Agent → 竞品监控 | 决定抓取哪些账号的爆款<br>• 监控10个 → 只学习头部<br>• 监控50个 → 学习中腰部爆款<br>• 监控100个 → 全行业监控 |
| **选题黑名单** | Strategy Agent | 策略Agent → 黑名单 | 过滤敏感话题<br>• 医疗IP禁止"治愈""根治"<br>• 金融IP禁止"稳赚""保本" |
| **平台策略** | Strategy Agent | 策略Agent → 平台策略 | 不同平台推荐不同选题<br>• 抖音偏好30-60秒短内容<br>• 视频号偏好2-3分钟深度内容 |
| **检索策略** | Memory Agent | 记忆Agent → 检索配置 | 影响素材匹配质量<br>• 语义相似度优先 → 理解话题内涵<br>• 新鲜度优先 → 优先新素材 |
| **标签体系** | Memory Agent | 记忆Agent → 标签管理 | 影响匹配精准度<br>• 情绪标签匹配 → 愤怒话题匹配愤怒故事<br>• 场景标签匹配 → 深夜话题匹配深夜场景 |

### 配置检查清单

- [ ] Strategy Agent - 四维权重已配置（建议：流量30%+变现30%+契合25%+成本15%）
- [ ] Strategy Agent - 竞品监控清单已添加（建议：10-20个对标账号）
- [ ] Strategy Agent - 选题黑名单已设置（行业敏感词）
- [ ] Memory Agent - 标签体系已配置（情绪/场景/认知标签）
- [ ] Memory Agent - 检索策略已选择（推荐：语义相似度优先）

## 场景二：🔥 仿写爆款

### 使用场景
IP复制竞品视频链接到输入框，选择风格（愤怒/冷静/幽默），点击"仿写"，AI生成保留爆款结构但替换为IP素材的文案。

### 调用Agent流程

```
┌────────────────────────────────────────────────────────────────┐
│                     仿写爆款调用链                              │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  用户输入：竞品视频链接 + 选择风格（愤怒/冷静/幽默）            │
│                    ↓                                           │
│  ① Remix Agent (核心)                                          │
│     ├─ 输入：竞品视频链接                                       │
│     ├─ 处理：                                                  │
│     │   • 解构竞品结构（钩子/故事/观点/CTA）                   │
│     │   • 提取钩子模式（"90%的人都不知道..."）                 │
│     │   • 分析情绪曲线（愤怒→希望→行动）                       │
│     │   • 识别论证结构（问题→案例→方法论）                     │
│     │   • 标记需替换元素（竞品案例/产品/数据）                  │
│     └─ 输出：结构模板 + 替换清单                                │
│         例：{                                                  │
│           hook: { pattern: "极端数字+否定", content: "竞品内容" },│
│           story: { structure: "失败→转折→成功", needReplace: true },│
│           ...                                                  │
│         }                                                      │
│                    ↓                                           │
│  ② Memory Agent (核心)                                         │
│     ├─ 输入：解构后的场景/情绪标签                              │
│     ├─ 处理：                                                  │
│     │   • 语义检索IP素材库                                      │
│     │   • 根据"深夜+绝望+翻盘"检索相关故事                      │
│     │   • 优先选择使用次数少的素材（新鲜度控制）                │
│     └─ 输出：3个可选的IP素材                                    │
│         例：{                                                  │
│           story_001: "2020年破产经历",                         │
│           story_002: "贵人相助故事",                           │
│           story_003: "团队解散危机"                            │
│         }                                                      │
│                    ↓                                           │
│  ③ Generation Agent (核心)                                     │
│     ├─ 输入：结构模板 + IP素材 + 选定风格                       │
│     ├─ 处理：                                                  │
│     │   • 保留爆款结构（钩子模式/情绪曲线）                     │
│     │   • 100%替换竞品案例为IP素材                              │
│     │   • 应用IP风格（口头禅/句长/修辞）                        │
│     │   • 应用选定情绪风格（愤怒/冷静/幽默）                    │
│     │   • 去AI化（插入口语瑕疵）                                │
│     └─ 输出：完整文案（分段展示）                               │
│                    ↓                                           │
│  ④ Compliance Agent (统一直出)                                  │
│     ├─ 输入：生成的文案                                         │
│     ├─ 处理：                                                  │
│     │   • 原创度检测（文本重复率<25%）                          │
│     │   • 结构相似度检测（<40%）                                │
│     │   • 敏感词检查                                            │
│     └─ 输出：审核结果（通过/警告/失败）                         │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 关键配置项

| 配置项 | 所在Agent | 配置路径 | 对场景的影响 |
|--------|-----------|----------|--------------|
| **解构规则** | Remix Agent | 重组Agent → 解构规则 | 决定提取竞品哪些要素<br>• 只开"钩子模式" → 只学开头<br>• 全开 → 完整复刻结构 |
| **杂交策略** | Remix Agent | 重组Agent → 重组规则 | 决定如何组合<br>• Best of Breed → 多个竞品各取最优<br>• 单一解构 → 深度复制一个爆款 |
| **原创度阈值** | Remix Agent | 重组Agent → 原创度保障 | 控制相似度<br>• 文本<25% + 结构<40% → 安全但可能失去爆款感<br>• 文本<15% + 结构<30% → 更原创但更不像 |
| **强制替换规则** | Remix Agent | 重组Agent → 强制替换 | 确保100%替换竞品元素<br>• 案例 → IP故事<br>• 产品 → IP产品<br>• 数据 → IP数据 |
| **素材分级/使用限制** | Memory Agent | 记忆Agent → 使用限制 | 控制素材使用频次<br>• 核心素材限3次 → 防止过度消费<br>• 常规素材限10次 → 平衡新鲜度 |
| **风格训练集** | Generation Agent | 生成Agent → 风格训练 | 决定仿写后"像不像"IP<br>• 50条S级文案 → 基础风格<br>• 定期更新 → 风格与时俱进 |
| **风格强度** | Generation Agent | 生成Agent → 风格强度 | 控制模仿程度<br>• 80% → 推荐值，既像又有新意<br>• 100% → 完全模仿，可能缺乏创新 |
| **情绪曲线模板** | Generation Agent | 生成Agent → 情绪曲线 | 决定文案节奏<br>• 愤怒-希望型 → 解决方案类<br>• 好奇-震惊型 → 揭秘类 |
| **去AI化参数** | Generation Agent | 生成Agent → 去AI化 | 增加真实感<br>• 每100字2处口语瑕疵<br>• 打破对仗，增加具体时间锚点 |
| **三级敏感词库** | Compliance Agent | 合规Agent → 敏感词库 | 确保合规安全<br>• 红线严格阻断<br>• 黄线警告提示<br>• 蓝线自动替换 |

### 配置检查清单

- [ ] Remix Agent - 解构规则已开启（建议：钩子+情绪+论证全开）
- [ ] Remix Agent - 杂交策略已选择（建议：Best of Breed）
- [ ] Remix Agent - 原创度阈值已设置（建议：文本<25%，结构<40%）
- [ ] Memory Agent - 素材使用限制已配置（建议：核心3次/常规10次）
- [ ] Generation Agent - 风格训练集已上传（建议：50条S级文案）
- [ ] Generation Agent - 风格强度已设置（建议：80%）
- [ ] Generation Agent - 口头禅已配置（高频/中频/低频）
- [ ] Generation Agent - 情绪曲线模板已选择（根据IP类型）
- [ ] Compliance Agent - 敏感词库已配置（行业特定）

## 场景三：🎙️ 语音输入想法

### 使用场景
IP按住麦克风按钮，说出自己的想法（如"我想讲讲2020年那次现金流危机"），AI自动扩展成完整文案。

### 调用Agent流程

```
┌────────────────────────────────────────────────────────────────┐
│                     语音输入调用链                              │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  用户输入：语音（按住麦克风说话）                               │
│  例："我想讲讲2020年那次现金流危机，真的太痛苦了..."           │
│                    ↓                                           │
│  ① ASR语音识别 (Whisper API)                                   │
│     ├─ 输入：音频流                                           │
│     ├─ 处理：语音转文字 + 时间戳 + 语气识别                    │
│     └─ 输出：文字稿 + 情绪标记（愤怒/焦虑/平静）               │
│         例："我想讲讲2020年那次现金流危机真的太痛苦了"         │
│                    ↓                                           │
│  ② Memory Agent (核心)                                         │
│     ├─ 输入：ASR文字 + 情绪标签                                │
│     ├─ 处理：                                                  │
│     │   • 语义理解：识别核心意图"现金流危机"                   │
│     │   • 情绪匹配：匹配"绝望+痛苦"情绪标签                     │
│     │   • 场景匹配：匹配"深夜/办公室"场景标签                   │
│     │   • 检索素材库：找到"2020破产经历"等相关故事              │
│     │   • 新鲜度控制：优先推荐使用次数少的素材                  │
│     └─ 输出：3-5个相关素材 + 情绪标签 + 场景标签               │
│                    ↓                                           │
│  ③ Generation Agent (核心)                                     │
│     ├─ 输入：用户原始文字 + 检索到的素材 + 情绪标签             │
│     ├─ 处理：                                                  │
│     │   • 基于素材扩展完整故事（从碎片到完整叙事）              │
│     │   • 应用IP风格（口头禅/句长/修辞习惯）                    │
│     │   • 自动选择合适的情绪曲线模板                            │
│     │   • 植入变现钩子（自然过渡）                              │
│     │   • 去AI化处理（口语瑕疵/具体时间锚点）                   │
│     └─ 输出：完整文案（分段：钩子/故事/观点/CTA）               │
│                    ↓                                           │
│  ④ Remix Agent (可选，根据配置)                                │
│     ├─ 输入：生成的文案                                        │
│     ├─ 处理：如果文案结构松散，应用爆款结构模板优化             │
│     └─ 输出：结构优化后的文案                                   │
│                    ↓                                           │
│  ⑤ Compliance Agent (统一直出)                                  │
│     ├─ 输入：最终文案                                          │
│     └─ 输出：审核结果                                           │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 关键配置项

| 配置项 | 所在Agent/服务 | 配置路径 | 对场景的影响 |
|--------|----------------|----------|--------------|
| **语音识别模型** | ASR服务 | 系统配置 | 影响转写准确率<br>• Whisper API → 支持多语言、口音<br>• 可配置专业术语词库 |
| **检索策略** | Memory Agent | 记忆Agent → 检索配置 | 影响素材匹配质量<br>• 语义相似度优先 → 理解意图而非关键词<br>• 多样性优先 → 提供不同角度的素材 |
| **情绪标签体系** | Memory Agent | 记忆Agent → 标签管理 | 影响情绪匹配<br>• 愤怒/焦虑/绝望/希望/爽感<br>• ASR识别的情绪自动匹配对应标签 |
| **场景标签体系** | Memory Agent | 记忆Agent → 标签管理 | 影响场景匹配<br>• 深夜/办公室/会议室/舞台<br>• 时间关键词自动匹配场景 |
| **新鲜度权重** | Memory Agent | 记忆Agent → 检索配置 | 控制素材新鲜度<br>• 权重高 → 优先新素材<br>• 权重低 → 优先经典素材 |
| **风格训练集** | Generation Agent | 生成Agent → 风格训练 | 决定扩展后的文案"像不像"IP |
| **风格强度** | Generation Agent | 生成Agent → 风格强度 | 控制扩展程度<br>• 80% → 基于素材发挥<br>• 100% → 严格按素材风格 |
| **情绪曲线模板** | Generation Agent | 生成Agent → 情绪曲线 | 自动选择或手动选择<br>• ASR识别愤怒 → 自动选"愤怒-希望型" |
| **去AI化参数** | Generation Agent | 生成Agent → 去AI化 | 保持口语化<br>• 保留ASR识别的口语习惯<br>• 插入口头禅 |
| **变现植入规则** | Generation Agent | 生成Agent → 变现规则 | 自然植入产品<br>• 只在"解决方案"段落出现<br>• 先讲痛点再出产品 |

### 配置检查清单

- [ ] ASR服务 - 语音识别API已配置（Whisper API Key）
- [ ] Memory Agent - 语义检索策略已选择（建议：语义相似度优先）
- [ ] Memory Agent - 标签体系已配置（情绪标签+场景标签）
- [ ] Memory Agent - 新鲜度权重已设置（建议：20%）
- [ ] Generation Agent - 风格训练集已上传
- [ ] Generation Agent - 情绪曲线模板已配置（多种情绪类型）
- [ ] Generation Agent - 变现植入规则已设置

## 🎯 配置建议优先级

### 第一阶段（MVP必须）
1. **Strategy Agent** - 竞品监控清单（决定选题来源）
2. **Memory Agent** - 标签体系（决定素材匹配）
3. **Generation Agent** - 风格训练集（决定内容质量）
4. **Compliance Agent** - 敏感词库（决定安全性）

### 第二阶段（优化体验）
5. **Strategy Agent** - 四维评分权重（优化选题质量）
6. **Remix Agent** - 原创度阈值（平衡仿写与原创）
7. **Generation Agent** - 口头禅/句长偏好（增强IP感）
