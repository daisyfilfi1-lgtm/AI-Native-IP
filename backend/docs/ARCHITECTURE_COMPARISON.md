# 架构对比：旧版 vs 新版

## 旧版架构

```
┌─────────────────────────────────────────────────────────────┐
│                      用户输入链接                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│              competitor_text_extraction.py                   │
│                                                              │
│  1. 短链解析（仅支持抖音）                                    │
│  2. TikHub API 尝试                                          │
│     └─ 失败？返回空字符串                                     │
│  3. （可选）yt-dlp 备选                                      │
│     └─ 失败？返回空字符串                                     │
│  4. 兜底：build_fallback_text()                              │
│     └─ 返回 URL 信息（质量极差）                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                      返回提取结果                            │
│         通常只有视频ID或空字符串，无法有效仿写                 │
└─────────────────────────────────────────────────────────────┘

问题：
❌ TikHub 失败时没有备选方案
❌ 兜底方案质量太差
❌ 没有错误分类和提示
❌ 不支持手动输入
```

## 新版架构

```
┌─────────────────────────────────────────────────────────────┐
│                      用户输入链接                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│              link_resolver.py (新增)                         │
│                                                              │
│  支持平台：                                                   │
│  ✓ 抖音 (v.douyin.com)                                       │
│  ✓ 小红书 (xhslink.com)                                      │
│  ✓ 快手 (v.kuaishou.com)                                     │
│  ✓ B站 (b23.tv)                                              │
│                                                              │
│  输出：原始URL、解析后URL、平台、视频ID、错误信息               │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│              text_extractor.py (新增)                        │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 策略1: TikHub API                                    │   │
│  │ • 抖音 Web 单条接口                                   │   │
│  │ • Hybrid 通用接口                                     │   │
│  │ • 提取 desc/title/comments                           │   │
│  │                                     成功 ✓           │   │
│  │                                     失败 ↓           │   │
│  └─────────────────────────────────────────────────────┘   │
│                            ↓                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 策略2: Web 爬取                                      │   │
│  │ • 抖音 API 端点 + 移动端页面                          │   │
│  │ • 小红书 SSR 数据解析                                 │   │
│  │ • 快手/B站 页面解析                                   │   │
│  │                                     成功 ✓           │   │
│  │                                     失败 ↓           │   │
│  └─────────────────────────────────────────────────────┘   │
│                            ↓                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 策略3: yt-dlp (可选)                                 │   │
│  │ • 支持 1000+ 站点                                     │   │
│  │ • 元数据提取                                          │   │
│  │                                     成功 ✓           │   │
│  │                                     失败 ↓           │   │
│  └─────────────────────────────────────────────────────┘   │
│                            ↓                                │
│              返回明确错误信息 + 元数据                        │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│              competitor_text_extraction.py (简化)            │
│                                                              │
│  调用统一服务，返回结构化结果                                  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                     提取结果处理                             │
│                                                              │
│  成功？ ──→ 继续仿写流程                                     │
│    ↓                                                         │
│  失败？ ──→ 返回分类错误信息                                 │
│                                                              │
│  前端显示：                                                   │
│  ✓ 错误类型提示                                              │
│  ✓ 针对性解决方案                                            │
│  ✓ 「手动粘贴文案」按钮                                      │
└─────────────────────────────────────────────────────────────┘
                            ↓ (手动输入)
┌─────────────────────────────────────────────────────────────┐
│                   手动输入模式                               │
│                                                              │
│  用户粘贴文案 ──→ 跳过链接提取 ──→ 直接仿写                  │
└─────────────────────────────────────────────────────────────┘

优势：
✅ 多策略确保高成功率
✅ 完善的错误处理和分类
✅ 手动输入后备方案
✅ 支持更多平台
✅ 结构化结果和元数据
```

## 关键改进对比

| 维度 | 旧版 | 新版 |
|-----|------|------|
| **短链解析** | 仅抖音 | 抖音、小红书、快手、B站 |
| **提取策略** | 单一 TikHub | TikHub + Web + yt-dlp |
| **失败兜底** | 返回 URL 文本 | 返回明确错误 |
| **错误处理** | 无分类 | 6+ 种错误类型 |
| **手动输入** | 不支持 | 支持 |
| **平台支持** | 抖音为主 | 4+ 平台 |
| **代码结构** | 混合逻辑 | 模块化设计 |
| **可扩展性** | 差 | 优秀 |

## 代码量对比

| 文件 | 旧版 | 新版 |
|-----|------|------|
| competitor_text_extraction.py | ~350 行 | ~60 行 |
| link_resolver.py | 无 | ~280 行 |
| text_extractor.py | 无 | ~550 行 |
| **总计** | ~350 行 | ~890 行 |

虽然代码量增加，但：
- 模块职责更清晰
- 可测试性更强
- 可维护性更好
- 扩展性更好

## 成功率对比（预估）

| 场景 | 旧版 | 新版 |
|-----|------|------|
| TikHub 配置正常 | 85% | 90% |
| TikHub 未配置 | 10% | 60% |
| TikHub 限流 | 20% | 75% |
| 特殊链接（短链等） | 40% | 85% |
| **综合成功率** | **~40%** | **~80%** |

（手动输入兜底可再提升 15-20% 至 **~95%**）
