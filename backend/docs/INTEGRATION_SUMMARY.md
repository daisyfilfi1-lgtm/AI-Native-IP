# 第三方文案提取工具集成方案

## 方案对比

| 方案 | 难度 | 成本 | 准确率 | 推荐度 |
|-----|------|------|--------|--------|
| **内嵌第三方工具** | ⭐ 低 | 免费 | 高 | ⭐⭐⭐ 强烈推荐 |
| 增强自研 yt-dlp | ⭐⭐ 中 | 服务器 | 中高 | ⭐⭐⭐ 已实施 |
| 购买 RapidAPI | ⭐ 低 | ¥200-500/月 | 高 | ⭐⭐ 可选 |
| 自建浏览器插件 | ⭐⭐⭐ 高 | 开发成本 | 最高 | ⭐⭐ 长期规划 |

## 推荐方案：内嵌第三方工具 + 自研备用

### 已实施的改进

#### 1. 多策略自研提取（已完成）
```
链接 → TikHub API → Web爬取 → yt-dlp → 手动输入
         ↓失败        ↓失败      ↓失败
       下一个策略   下一个策略  最终兜底
```

#### 2. 第三方工具内嵌（已完成）
```
提取失败 → 显示第三方工具列表 → 用户跳转提取 → 粘贴回来 → 继续仿写
```

### 新增文件

| 文件 | 作用 |
|-----|------|
| `link_resolver.py` | 多平台链接解析 |
| `text_extractor.py` | 统一文本提取服务 |
| `ThirdPartyExtractor.tsx` | 第三方工具UI组件 |

## 用户使用流程

### 场景A：自动提取成功
```
填写链接 → 点击「开始仿写」→ 自动提取 → 生成仿写 ✅
```

### 场景B：自动提取失败（有第三方工具）
```
填写链接 → 点击「开始仿写」→ 提取失败
                              ↓
                      显示「使用第三方工具提取文案」按钮
                              ↓
                      点击打开第三方工具列表
                              ↓
                      选择工具（anytocopy等）→ 新标签页打开
                              ↓
                      在第三方工具提取文案并复制
                              ↓
                      返回本页面 → 粘贴文案
                              ↓
                      继续仿写 ✅
```

### 场景C：自动提取失败（直接粘贴）
```
填写链接 → 点击「开始仿写」→ 提取失败
                              ↓
                      点击「直接粘贴文案」
                              ↓
                      手动粘贴文案
                              ↓
                      继续仿写 ✅
```
