# 仿写流程修复报告 - 文本提取增强

## 问题背景

用户反馈："爆款指定链接填写后，仿写走不通"

根本原因：**链接 → 文本提取成功率低**，导致仿写流程无法获取有效的竞品文案。

## 修复方案概览

### 核心改进

采用**多策略融合**架构，整合多种文本提取方案：

1. **TikHub API** - 官方数据源，最准确
2. **Web 爬取** - 不依赖第三方 API，自主可控
3. **yt-dlp** - 通用兜底方案

### 架构升级

```
修复前（单一策略）
─────────────────────
链接 → TikHub API → 成功/失败
           ↓
       失败时返回 URL 文本（质量极差）

修复后（多策略融合）
─────────────────────
链接 → 解析 → TikHub → 成功 ✓
          ↓ 失败
       Web 爬取 → 成功 ✓
          ↓ 失败
       yt-dlp → 成功 ✓
          ↓ 失败
       返回明确错误 + 手动输入选项
```

## 提取策略详解

### 策略1: TikHub API

**适用：** 已配置 TIKHUB_API_KEY

**流程：**
1. 抖音链接 → 优先使用 Web 单条作品接口
2. 其他平台 → 使用 Hybrid 通用接口
3. 解析响应，提取 desc/title/comments

**优点：**
- 数据准确完整
- 包含评论等附加信息

**缺点：**
- 依赖第三方 API
- 需要配置 API Key

### 策略2: Web 爬取

**适用：** TikHub 失败或未配置

**支持平台：**

| 平台 | 爬取方式 |
|-----|---------|
| 抖音 | API 端点 + 移动端页面 |
| 小红书 | SSR 数据解析 |
| 快手 | 移动端页面 |
| B站 | 桌面端页面 |

**优点：**
- 自主可控
- 无需 API Key

**缺点：**
- 可能受反爬限制
- 页面结构变更需要更新

### 策略3: yt-dlp

**适用：** 以上都失败，且已安装 yt-dlp

**配置：**
```bash
pip install yt-dlp
export REMIX_YTDLP_FALLBACK=1
```

**优点：**
- 支持 1000+ 站点
- 稳定可靠

**缺点：**
- 需要额外安装
- 提取速度较慢

## 手动输入后备

当所有自动提取策略都失败时，用户可以选择手动输入：

**使用流程：**
1. 点击"手动粘贴文案"
2. 复制视频标题和文案
3. 粘贴到输入框
4. 点击"使用此文案"

**后端处理：**
- 识别 `[MANUAL_TEXT]` 前缀
- 跳过链接解析和提取
- 直接使用用户提供的内容

## 错误处理改进

### 错误分类

| 错误类型 | 触发条件 | 用户提示 |
|---------|---------|---------|
| API 未配置 | TIKHUB_API_KEY 缺失 | "请联系管理员配置" |
| API 权限不足 | 403 错误 | "请检查 TikHub 权限" |
| 视频不存在 | 404 错误 | "视频已被删除" |
| 提取超时 | 网络问题 | "请稍后重试" |
| Web 爬取失败 | 反爬/结构变更 | "建议手动粘贴文案" |

### 调试信息

提取失败时，后端返回：
```json
{
  "id": "gen_remix_extract_failed",
  "status": "failed",
  "error": "用户友好的错误信息",
  "details": {
    "stage": "text_extraction",
    "url": "链接前100字符",
    "raw_error": "原始错误信息",
    "metadata": {
      "platform": "douyin",
      "resolved_url": "解析后的链接"
    }
  }
}
```
