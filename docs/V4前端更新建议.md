# V4前端更新建议

## 🎯 当前问题

前端在策略Agent页面调用的是旧API (`/content/topics/recommend`)，虽然后端已经整合了V4逻辑，但前端没有显示V4特有的字段（如竞品来源、重构置信度等）。

## 💡 建议的改进

### 1. 显示竞品来源信息

在选题卡片上增加竞品来源标识：

```tsx
// 在 TopicRecommendation 类型中添加 V4 字段
type TopicRecommendation = {
  title: string;
  score: number;
  reason: string;
  trend: string;
  viral_elements: string[];
  // V4 新增字段
  _v4_data?: {
    original_title: string;
    is_remixed: boolean;
    remix_confidence: number;
    competitor_author: string;
    competitor_play_count: number;
    content_type: string;
    content_angle: string;
  };
};
```

### 2. 更新选题卡片UI

```tsx
// 在策略Agent页面的选题卡片中
{topic._v4_data?.is_remixed && (
  <div className="flex items-center gap-2 text-xs text-blue-600 mt-1">
    <Badge variant="secondary" className="bg-blue-50">
      竞品重构
    </Badge>
    <span>参考: {topic._v4_data.competitor_author}</span>
    <span>({(topic._v4_data.competitor_play_count / 10000).toFixed(1)}万播放)</span>
    <span className="text-gray-400">
      置信度: {Math.round(topic._v4_data.remix_confidence * 100)}%
    </span>
  </div>
)}
```

### 3. 可选：直接调用V4 API

如果希望使用V4专门的API（支持更多参数）：

```typescript
// lib/api.ts 中添加新方法
async recommendTopicsV4(
  ipId: string,
  limit = 12,
  strategy: 'competitor_first' | 'competitor_only' | 'hybrid' = 'competitor_first'
): Promise<{
  ip_id: string;
  total_count: number;
  strategy: string;
  topics: RecommendedTopicV4[];
  competitor_stats: {
    competitor_count: number;
    video_count: number;
    avg_play_count: number;
  };
}> {
  const response = await this.client.get('/strategy/v4/topics/recommend', {
    params: { ip_id: ipId, limit, strategy },
  });
  return response.data;
}
```

### 4. 显示竞品统计信息

在策略Agent页面顶部添加竞品统计：

```tsx
<div className="flex gap-4 mb-4 text-sm text-gray-600">
  <span>竞品账号: {competitorStats.competitor_count}</span>
  <span>参考视频: {competitorStats.video_count}</span>
  <span>平均播放: {(competitorStats.avg_play_count / 10000).toFixed(1)}万</span>
</div>
```

## 🔧 最小改动方案

如果希望最小改动，只需修改一处：**显示竞品来源**

在 `frontend/app/agents/strategy/page.tsx` 中，找到选题卡片渲染部分（约450-500行），添加：

```tsx
{topic._v4_data?.is_remixed && (
  <div className="mt-2 text-xs text-gray-500">
    <span className="text-blue-600">🔗 竞品来源: {topic._v4_data.competitor_author}</span>
    <span className="ml-2">{(topic._v4_data.competitor_play_count / 10000).toFixed(1)}万播放</span>
  </div>
)}
```

## 📊 预期效果

改进后，用户可以看到：
1. 选题来自哪个竞品账号
2. 原竞品的播放量（证明这个选题有效）
3. 是否经过智能重构
4. 重构的置信度

这样可以**增强用户对推荐选题的信任感**。

// types/index.ts

  title: string;

export interface TopicRecommendationV4Response {
