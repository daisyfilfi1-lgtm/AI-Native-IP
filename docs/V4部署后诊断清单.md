# V4选题推荐系统 - 部署后诊断清单

## 🔍 问题现象

前端没有看到竞品爆款数据的变化，仍然显示旧的推荐内容。

## ❓ 可能原因

### 1. 数据库中没有竞品数据 ⭐最常见

**检查方法**：
```sql
-- 连接到你的PostgreSQL数据库
-- 检查竞品账号
SELECT COUNT(*) FROM competitor_accounts WHERE ip_id = 'xiaomin';

-- 检查竞品视频
SELECT COUNT(*) FROM competitor_videos cv
JOIN competitor_accounts ca ON cv.competitor_id = ca.competitor_id
WHERE ca.ip_id = 'xiaomin';
```

**如果数量为0，请执行**：
```bash
# 在Railway或服务器上执行
psql $DATABASE_URL -f backend/scripts/setup_competitors_from_analysis.sql
psql $DATABASE_URL -f backend/scripts/seed_competitor_videos.sql
```

### 2. 后端代码未更新

**检查方法**：
```bash
# SSH进入Railway容器或服务器
cd backend

# 检查V4文件是否存在
ls -la app/services/topic_recommendation_v4.py
ls -la app/services/competitor_content_remixer.py
ls -la app/services/datasource/competitor_source.py

# 检查 content_generation_pipeline.py 是否已修改
grep -n "V4\|_fetch_v4_topics\|_convert_v4_to_legacy" app/services/content_generation_pipeline.py
```

**如果没有这些代码，需要重新部署**：
```bash
git pull origin master
```

### 3. 前端调用的是旧API

**检查方法**：
打开浏览器开发者工具 → Network → 查看请求：
- **旧API**: `POST /api/v1/content/topics/recommend` (Body: `{ip_id, count}`)
- **V4 API**: `GET /api/v1/strategy/v4/topics/recommend?ip_id=xiaomin&limit=6`

**前端当前调用的是旧API**，但旧API已经整合了V4逻辑，所以应该也能工作。

### 4. API响应中没有V4数据

**检查方法**：
直接测试API：
```bash
# 测试旧API（前端正在使用的）
curl -X POST https://your-api.com/api/v1/content/topics/recommend \
  -H "Content-Type: application/json" \
  -d '{"ip_id": "xiaomin", "count": 3}'

# 检查返回的topic中是否有 _v4_data 字段
```

**期望的响应结构**：
```json
{
  "ip_id": "xiaomin",
  "recommendations": [
    {
      "title": "在家带娃3年没收入，我用这个副业方法月入3万",
```

## ✅ 验证步骤

### 步骤1: 验证数据库数据

```bash
# 进入Railway容器的Shell或使用psql连接

# 1. 检查表是否存在
\dt competitor_accounts
\dt competitor_videos

# 2. 检查数据
SELECT 'Competitors' as table_name, COUNT(*) as count FROM competitor_accounts WHERE ip_id = 'xiaomin'
UNION ALL
SELECT 'Videos', COUNT(*) FROM competitor_videos cv
JOIN competitor_accounts ca ON cv.competitor_id = ca.competitor_id
WHERE ca.ip_id = 'xiaomin';
```


## 🔧 修复步骤

### 如果数据库中没有数据

```bash
# 1. 复制SQL文件到服务器
scp backend/scripts/*.sql railway:/app/backend/scripts/

# 2. 执行SQL
psql $DATABASE_URL -f backend/scripts/create_competitor_tables.sql
psql $DATABASE_URL -f backend/scripts/setup_competitors_from_analysis.sql
psql $DATABASE_URL -f backend/scripts/seed_competitor_videos.sql
```

### 如果后端代码未更新

```bash
# 1. 确保代码已推送
git log --oneline -3  # 应该包含V4相关提交

# 2. 在服务器上拉取最新代码
git pull origin master

# 3. 重启服务
systemctl restart your-app  # 或 railway up
```

### 如果API没有返回V4数据

检查 `content_generation_pipeline.py` 中的 `recommend_topics` 方法是否包含以下代码：
```python
# 尝试使用V4竞品爆款系统
try:
    v4_result = ...
    if v4_result and len(v4_result) > 0:
        normalized = self._convert_v4_to_legacy(v4_result)
        return {...}
except Exception as e:
    logging.warning(f"[V4] Failed...")

# 回退到LLM生成
return self._recommend_topics_llm(n)
```

复制粘贴以下命令到Railway Shell：

```bash
# 一键诊断
echo "=== V4 System Diagnostic ===" && \
echo "1. Database Tables:" && \
psql $DATABASE_URL -c "SELECT table_name FROM information_schema.tables WHERE table_name IN ('competitor_accounts', 'competitor_videos');" && \
echo && \
echo "2. Competitor Count:" && \
psql $DATABASE_URL -c "SELECT COUNT(*) as competitors FROM competitor_accounts WHERE ip_id = 'xiaomin';" && \
echo && \
echo "3. Video Count:" && \
psql $DATABASE_URL -c "SELECT COUNT(*) as videos FROM competitor_videos cv JOIN competitor_accounts ca ON cv.competitor_id = ca.competitor_id WHERE ca.ip_id = 'xiaomin';" && \
echo && \
echo "4. Recent Videos:" && \
psql $DATABASE_URL -c "SELECT cv.title, cv.play_count, ca.name FROM competitor_videos cv JOIN competitor_accounts ca ON cv.competitor_id = ca.competitor_id WHERE ca.ip_id = 'xiaomin' ORDER BY cv.play_count DESC LIMIT 3;"
```
