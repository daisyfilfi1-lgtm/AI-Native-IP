3. 点击 "Shell" 标签页

## 🔧 手动执行（如果脚本失败）

如果自动脚本执行失败，可以手动执行：

### 1. 更新代码

```bash
cd /app
git pull origin master
```

### 2. 执行SQL

```bash
# 创建表
psql $DATABASE_URL -f backend/scripts/create_competitor_tables.sql

# 插入竞品账号
psql $DATABASE_URL -f backend/scripts/setup_competitors_from_analysis.sql

# 插入示例视频
psql $DATABASE_URL -f backend/scripts/seed_competitor_videos.sql
```

### 3. 验证数据

```bash
# 检查数据
psql $DATABASE_URL -c "SELECT COUNT(*) FROM competitor_accounts WHERE ip_id = 'xiaomin'"
psql $DATABASE_URL -c "SELECT COUNT(*) FROM competitor_videos cv JOIN competitor_accounts ca ON cv.competitor_id = ca.competitor_id WHERE ca.ip_id = 'xiaomin'"
```

### 4. 重启服务

```bash
# Railway会自动重启，或者手动：
kill 1  # 让容器重启
```

## 🐛 常见问题

### 问题1: git pull 失败

**现象**: `fatal: unable to access...`

**解决**: 
```bash
# 强制更新
git fetch origin
git reset --hard origin/master
```

### 问题2: psql 命令不存在

**现象**: `psql: command not found`

**解决**:
```bash
# 安装postgresql-client
apt-get update && apt-get install -y postgresql-client

# 或者使用Python
python3 -c "
import os
import psycopg2
conn = psycopg2.connect(os.environ['DATABASE_URL'])
cursor = conn.cursor()
cursor.execute(open('backend/scripts/create_competitor_tables.sql').read())
conn.commit()
"
```

### 问题3: 数据库权限错误

**现象**: `permission denied`

**解决**: 检查DATABASE_URL是否有写权限

### 问题4: API返回空数据

**现象**: 验证脚本显示 "没有返回选题"

**解决**:
1. 检查后端日志：`railway logs`
2. 检查数据库是否有数据
3. 检查IP是否存在：`SELECT * FROM ip WHERE ip_id = 'xiaomin'`
