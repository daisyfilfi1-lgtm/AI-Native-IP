# 真正的LLM语义改写

## 核心区别

### ❌ 表面替换（旧方案）
```
原标题："90后宝妈靠副业月入过万"
表面替换："90后UI设计靠副业月入过万"
问题：只是替换关键词，没有理解为什么原标题能火
```

### ✅ 语义改写（新方案）
```
原标题："90后宝妈靠副业月入过万"
LLM分析：
- 核心吸引力：普通人也能赚大钱的希望
- 结构要素：身份标签(90后宝妈)+金钱结果(月入过万)+干货承诺
- 情绪触发：希望+羡慕
- 受众心理："我也可以"

语义改写："UI设计师靠AI接私单月入3万，分享我的3个获客渠道"
- 保留结构：身份+收入+干货
- 完全IP化：换成设计师领域的真实场景
- 自然流畅：像IP本人说的话
```

## Prompt设计

### 三步提示法

**第一步：分析爆款逻辑**
```
请分析原标题为什么能火：
1. 核心吸引力：抓住了什么痛点/欲望/好奇心？
2. 结构要素：数字？身份标签？对比？悬念？
3. 情绪触发：焦虑？希望？愤怒？好奇？
4. 受众心理：读者看到后会想什么？
```

**第二步：IP人设画像**
```
- IP名称、昵称、简介
- 擅长领域、热爱领域
- 内容方向、目标受众
- 语气风格、常用词汇
```

**第三步：语义改写要求**
```
1. 不要逐字替换，理解吸引力逻辑后重新创作
2. 保留爆款结构（数字位置、对比结构等）
3. 完全IP化：用IP的专业领域、经验重构内容
4. 语言风格化：像IP本人说的话
```

## 输出格式

```json
{
    "analysis": "原标题爆款逻辑分析",
    "title": "改写后的完整标题",
    "hook": "改写后的hook",
    "body": "改写后的body",
    "strategy_note": "改写策略说明"
}
```

## 降级策略

当LLM不可用时，使用规则改写：
1. 识别原标题结构（身份+结果+干货）
2. 替换身份为IP专业领域
3. 调整内容为IP相关内容
4. 保持语气自然

| 32岁，我终于活成了别人羡慕的样子 | 32岁设计师，我终于活成了别人羡慕的样子 | 32岁，从月薪5千到年入50万，我的设计师转型之路 |
| 千万别买这5种家电，我被骗了3万块 | 千万别用这5种AI工具，我浪费了3万块 | 新手设计师避坑：这5个AI误区让我多走3年弯路 |

## 优势

1. **理解爆款逻辑**：不只是替换，而是理解为什么火
2. **完全IP化**：内容是IP的专业领域，不是硬套
3. **自然流畅**：像IP本人说的话，不生硬
4. **保留结构**：数字、对比等爆款结构保留
5. **可解释**：输出中包含分析字段，知道为什么这样改
