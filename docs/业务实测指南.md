# 业务实测指南

用于验证 Phase 1「素材进入 → Memory Agent 结构化」端到端流程可跑通。

---

## 一、环境准备

### 1. 后端（Railway 部署）

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | PostgreSQL，由 Railway Postgres 自动注入 |
| `FEISHU_APP_ID` / `FEISHU_APP_SECRET` | 可选。飞书同步时使用；也可在管理后台「飞书应用凭证」中填写 |
| `BAIDU_PAN_ACCESS_TOKEN` | 可选。百度网盘同步；也可在管理后台填写（见 `docs/BAIDU_PAN.md`） |

### 2. 前端（本地或单独部署）

| 变量 | 说明 |
|------|------|
| `NEXT_PUBLIC_API_URL` | 后端基地址，如 `https://xxx.up.railway.app/api/v1` |

本地开发时若不设置，默认请求 `/api/v1`（需通过 Next 代理或同源部署）。

### 3. CORS

后端已配置 CORS 允许任意来源，前端单独部署时可正常跨域请求。

## 二、推荐素材录入 URL（text/document）

`requests.get(url)` 拉取原始响应文本，适合 **纯文本 / Markdown** 类 URL：

| 类型 | 示例 |
|------|------|
| Raw Markdown | `https://raw.githubusercontent.com/user/repo/main/README.md` |
| 纯文本 API | 返回 `Content-Type: text/plain` 的接口 |
| 公开 .txt | 任意返回纯文本的 HTTP URL |

**不推荐**：普通 HTML 网页（会包含大量标签，分块效果差）。

### 快速测试用示例 URL

```
https://raw.githubusercontent.com/microsoft/TypeScript/main/README.md
```

或任意可公网访问的 `.md` / `.txt` 文件 raw 地址。

## 三、端到端实测流程

### 步骤 1：创建 IP

```bash
curl -X POST "https://你的域名/api/v1/ip" \
  -H "Content-Type: application/json" \
  -d '{"ip_id":"test_001","name":"测试IP","owner_user_id":"admin"}'
```

或在前端 **IP 管理** → 新建 IP。

### 步骤 2：素材录入（URL）

```bash
curl -X POST "https://你的域名/api/v1/memory/ingest" \
  -H "Content-Type: application/json" \
  -d '{
    "ip_id":"test_001",
    "source_type":"text",
    "source_url":"https://raw.githubusercontent.com/microsoft/TypeScript/main/README.md",
    "title":"TypeScript README"
  }'
```

返回 `ingest_task_id`，可轮询状态：

```bash
curl "https://你的域名/api/v1/memory/ingest/{task_id}"
```

**本地上传（文档 / 音频 / 视频）**：需配置 S3 兼容对象存储（见 `backend/.env.example` 中 `STORAGE_*`）。

1. `POST /api/v1/memory/upload`（`multipart/form-data`：`ip_id`、`file`）得到 `file_id`。
2. `POST /api/v1/memory/ingest`，`source_type` 填 `document`/`text` 或 `audio`/`video`，`local_file_id` 填上一步的 `file_id`（可与 `source_url` 二选一或优先本地上传）。

### 步骤 3：验证素材入库

```bash
curl "https://你的域名/api/v1/memory/assets?ip_id=test_001&limit=20"
```

应返回 `items` 数组，含 `asset_id`、`title`、`content_snippet` 等。

### 步骤 4：语义检索（关键词）

```bash
curl -X POST "https://你的域名/api/v1/memory/retrieve" \
  -H "Content-Type: application/json" \
  -d '{"ip_id":"test_001","query":"TypeScript","top_k":5}'
```

应返回匹配的素材片段（当前为 `content ILIKE` 关键词匹配，未接入向量库）。

### 步骤 5：待办标签 / 确认标签

```bash
# 列出待复核
curl "https://你的域名/api/v1/memory/pending-labels?ip_id=test_001"

# 确认标签（asset_id 从上面获取）
curl -X POST "https://你的域名/api/v1/memory/labels/{asset_id}" \
  -H "Content-Type: application/json" \
  -d '{"ip_id":"test_001","confirmed_labels":{"emotion":"中性","scene":"技术"}}'
```

确认后，该素材不再出现在待复核列表。

### 步骤 6：飞书同步（可选）

若已配置飞书凭证（环境变量或管理后台）：

1. `GET /api/v1/integrations/feishu/spaces` 查看可同步空间
2. `POST /api/v1/integrations/feishu/sync`，Body: `{"ip_id":"test_001","space_id":"可选"}`

### 步骤 6b：百度网盘同步（可选）

若已配置 `BAIDU_PAN_ACCESS_TOKEN`（或管理后台百度网盘凭证），详见 **`docs/BAIDU_PAN.md`**：

1. `GET /api/v1/integrations/baidu/config`
2. `POST /api/v1/integrations/baidu/sync`，Body 示例：`{"ip_id":"test_001","remote_path":"/","recursive":true}`

### 步骤 7：Memory 配置

`GET /api/v1/config/memory?ip_id=test_001` 与  
`POST /api/v1/config/memory` 保存标签/检索/使用限制配置（见 API 文档）。

自动打标术语表在 `tag_config.tag_categories` 中按 IP 自定义；可参考 **`docs/TAG_TAXONOMY_REFERENCE.md`**（内容语义四维与示例 JSON，非强制）。

## 四、前端实测流程

1. **IP 管理**：新建 IP（如 test_001）
2. **记忆 Agent → 素材录入**：选择类型 text/document，填写 URL，提交录入
3. **记忆 Agent → 语义检索**：输入 IP ID 和关键词，点击搜索
4. **记忆 Agent → 标签管理 → 待复核列表**：输入 IP ID，查看并确认标签
5. **记忆 Agent → 高级配置**：按 IP 保存检索策略、使用限制
6. **配置中心**：选择 IP，查看记忆 Agent 配置状态

## 五、API 速查

| 接口 | 方法 | 用途 |
|------|------|------|
| `/api/v1/ip` | GET | 列表 IP |
| `/api/v1/ip` | POST | 创建 IP |
| `/api/v1/ip/{ip_id}` | GET | 查询 IP |
| `/api/v1/memory/ingest` | POST | 素材录入 |
| `/api/v1/memory/ingest/{task_id}` | GET | 录入任务状态 |
| `/api/v1/memory/assets` | GET | 素材列表（验证入库） |
| `/api/v1/memory/retrieve` | POST | 语义检索（关键词） |
| `/api/v1/memory/pending-labels` | GET | 待复核标签 |
| `/api/v1/memory/labels/{asset_id}` | POST | 确认标签 |
| `/api/v1/config/memory` | GET/POST | Memory 配置 |
| `/api/v1/integrations/feishu/*` | - | 飞书同步 |
| `/api/v1/integrations/baidu/*` | - | 百度网盘同步 |

## 六、常见问题

| 问题 | 排查 |
|------|------|
| 素材录入选 text/document 报错 | 确认已提供 `source_url` 或先 `POST /memory/upload` 后传 `local_file_id`；URL 需可公网访问 |
| 检索无结果 | 确认该 IP 下已有素材（GET /memory/assets），且 query 在 content 中存在 |
| 待复核列表为空 | 可能该 IP 下素材均已确认过标签；或尚未录入任何素材 |
| 前端请求 CORS 失败 | 确认后端已部署含 CORS 配置的最新版本 |
