# IP 素材上传链路检查清单

本文档用于验证「素材 → 数据库」全链路的可连通性。

---

## 零、本地一键跑通（推荐）

若 Supabase/云端数据库在本机 DNS 解析失败，可用本地 PostgreSQL 快速跑通：

```bash
# 1. 启动本地 PostgreSQL
docker-compose up -d

# 2. 复制本地配置并填入 OPENAI_API_KEY
cp backend/.env.local.example backend/.env
# 编辑 backend/.env，填入 OPENAI_API_KEY

# 3. 执行数据库迁移
cd backend && python scripts/run_migrations.py

# 4. 启动后端
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

本地上传文件无需配置 S3：未配置 `STORAGE_*` 时自动使用 `backend/data/uploads`。

## 一、上传路径概览

| 路径 | 入口 | 适用场景 | 依赖 |
|------|------|----------|------|
| **URL 录入** | `POST /api/v1/memory/ingest` (source_url) | 纯文本/Markdown URL | DB、Embedding |
| **本地上传+录入** | `POST /api/v1/memory/upload` → `POST /api/v1/memory/ingest` (local_file_id) | 文档/音视频文件 | DB、Storage(S3)、Embedding、(音视频)Whisper |
| **多模态素材** | `POST /api/v1/multimodal/asset` | 图片/视频/音频理解 | DB、Vision Model |

## 二、核心链路依赖检查

### 1. 数据库 (PostgreSQL) — **必选**

- **环境变量**: `DATABASE_URL`
- **表**: `ip`, `ip_assets`, `ingest_tasks`, `file_objects`, `asset_vectors`
- **验证**:
  ```bash
  # 健康检查
  curl https://你的域名/health
  ```
  或执行迁移确保表存在：`001_init.sql`、`002_ingest_tasks.sql`、`004_storage_and_vectors.sql`

### 2. Embedding / 向量化 — **必选（语义检索）**

- **环境变量**: `OPENAI_API_KEY`、`OPENAI_BASE_URL`（或兼容端点）、`EMBEDDING_MODEL`
- **用途**: 素材向量化写入 `asset_vectors`，供 ` /memory/retrieve` 语义检索
- **无配置时**: 向量写入失败不会阻断主流程，素材仍会入库；`/memory/retrieve` 会回退到关键词 ILIKE 检索

### 3. 对象存储 (S3/OSS) — **仅本地上传需要**

- **环境变量**: `STORAGE_ENDPOINT`、`STORAGE_ACCESS_KEY`、`STORAGE_SECRET_KEY`、`STORAGE_BUCKET`
- **用途**: `POST /memory/upload` 上传文件
- **未配置时**: 自动使用本地 `backend/data/uploads`，无需 S3
- **禁用本地 fallback**: 设置 `STORAGE_LOCAL_DISABLED=true`

### 4. 语音转写 (Whisper) — **仅 audio/video 需要**

- **环境变量**: `OPENAI_TRANSCRIPTION_API_KEY`（或主 `OPENAI_API_KEY`）、`OPENAI_TRANSCRIPTION_BASE_URL`、`WHISPER_MODEL`
- **用途**: 音视频文件/URL 转文字后再入库
- **未配置**: 音视频会写入占位文案，提示需配置转写 API

### 5. Qdrant 向量库 — **可选**

- **环境变量**: `QDRANT_URL`、`QDRANT_API_KEY`（云端）
- **用途**: `/vector/search`、混合检索 (`hybrid_search`)
- **说明**: **素材录入 (ingest) 默认写入 PostgreSQL `asset_vectors`**，不会自动写入 Qdrant。若使用 `/vector/search` 或混合检索，需执行 `scripts/migrate_to_qdrant.py` 做一次迁移。

## 四、推荐端到端自测

### 方式 A：URL 录入（无需 Storage）

1. 创建 IP  
   ```bash
   curl -X POST "http://localhost:8000/api/v1/ip" -H "Content-Type: application/json" \
     -d '{"ip_id":"test_001","name":"测试IP","owner_user_id":"admin"}'
   ```

2. 素材录入（URL）  
   ```bash
   curl -X POST "http://localhost:8000/api/v1/memory/ingest" -H "Content-Type: application/json" \
     -d '{"ip_id":"test_001","source_type":"text","source_url":"https://raw.githubusercontent.com/microsoft/TypeScript/main/README.md","title":"TS README"}'
   ```

3. 轮询任务状态（替换 `{task_id}`）  
   ```bash
   curl "http://localhost:8000/api/v1/memory/ingest/{task_id}"
   ```

4. 验证入库  
   ```bash
   curl "http://localhost:8000/api/v1/memory/assets?ip_id=test_001&limit=20"
   ```

### 方式 B：本地上传 + 录入（需 Storage）

1. 上传文件  
   ```bash
   curl -X POST "http://localhost:8000/api/v1/memory/upload" \
     -F "ip_id=test_001" -F "file=@/path/to/your.txt"
   ```
   返回的 `file_id` 作为 `local_file_id`。

2. 发起录入  
   ```bash
   curl -X POST "http://localhost:8000/api/v1/memory/ingest" -H "Content-Type: application/json" \
     -d '{"ip_id":"test_001","source_type":"text","local_file_id":"<上一步的file_id>","title":"本地文本"}'
   ```

## 五、常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 404 IP 不存在 | IP 未创建 | 先 `POST /api/v1/ip` |
| 503 对象存储未配置 | 已设置 `STORAGE_LOCAL_DISABLED=true` 且未配 S3 | 去掉该设置，或配置 S3/OSS |
| 转写失败 | 未配置 Whisper API | 配置 `OPENAI_TRANSCRIPTION_API_KEY` 或避免 audio/video |
| 向量检索无结果 | Embedding 未配置或 Qdrant 未迁移 | 配置 Embedding；或执行 `migrate_to_qdrant.py` 后使用 `/vector/search` |
| 任务 FAILED | 查看 `error_message` | `GET /api/v1/memory/ingest/{task_id}` 返回错误详情 |
