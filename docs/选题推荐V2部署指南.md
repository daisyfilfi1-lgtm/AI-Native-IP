# 选题推荐 V2.0 部署指南

## 一、主要改进

### 问题修复
| 问题 | V1.0状态 | V2.0解决方案 |
|------|---------|-------------|
| TIKHUB失效返回空 | 返回空数组 | 内置爆款库兜底，确保>95%返回率 |
| 核心词过滤太严 | 匹配率~10% | 语义匹配+同义词扩展，匹配率>80% |
| 热点改写质量差 | 机械拼接 | 直接用高质量内置选题 |
| 内容类型单一 | 随机分布 | 按4-3-2-1矩阵分配 |

### 核心特性
1. **内置爆款库**：50+高质量选题，无需TIKHUB也能工作
2. **语义匹配**：基于sentence-transformers的相似度计算
3. **关键词同义词库**：支持模糊匹配
4. **四维评分**：相关度+热度+竞争度+转化度
5. **内容矩阵**：自动按40%/30%/20%/10%分配

## 二、部署步骤

### Step 1: 安装依赖（可选）

如果使用语义匹配功能，建议安装：
```bash
cd backend
pip install sentence-transformers
```

> 如果不安装，系统会自动回退到关键词匹配，效果略差但仍可用。

### Step 2: 注册路由

在 `backend/app/main.py` 中添加新路由：

```python
# 导入新路由
from app.routers import topic_recommendation_v2

# 注册路由（在原有路由之后添加）
app.include_router(
    topic_recommendation_v2.router,
    prefix="/api/v1",
    tags=["topic-recommendation-v2"]
)
```

### Step 3: 配置环境变量（可选）

V2.0不再强制依赖TIKHUB，但建议保留配置以获取更多数据源：

```bash
# TIKHUB配置（可选但推荐）
TIKHUB_API_KEY=your_api_key_here
TIKHUB_BASE_URL=https://api.tikhub.io
```

### Step 4: 测试API

启动服务后测试新接口：

```bash
# 测试新接口（推荐）
curl "http://localhost:8000/api/v1/strategy/v2/topics/recommend?ip_id=xiaomin1&limit=12"

# 测试内置库（无需TIKHUB）
curl "http://localhost:8000/api/v1/strategy/v2/topics/builtin?ip_id=xiaomin1&limit=12"

# 测试匹配算法
curl "http://localhost:8000/api/v1/strategy/v2/topics/match-test?ip_id=xiaomin1&topic_title=宝妈创业月入3万的方法"
```

## 三、API变更说明

### 新接口（推荐）

| 接口 | 说明 |
|------|------|
| `GET /api/v1/strategy/v2/topics/recommend` | 智能推荐V2（语义匹配+内置兜底） |
| `GET /api/v1/strategy/v2/topics/builtin` | 内置爆款库 |
| `GET /api/v1/strategy/v2/topics/match-test` | 匹配算法测试 |
| `GET /api/v1/strategy/v2/topics/recommend-compat` | 兼容旧版格式 |

### 旧接口（保持兼容）

| 接口 | 说明 | 状态 |
|------|------|------|
| `GET /api/v1/creator/topics/recommended` | 原有推荐接口 | 保持兼容 |
| `GET /api/v1/strategy/topics/recommend` | V1推荐接口 | 保持兼容 |

### 前端迁移建议

**方案1：渐进迁移（推荐）**
```javascript
// 先使用兼容接口，格式与旧版一致
const topics = await fetch('/api/v1/strategy/v2/topics/recommend-compat?ip_id=xiaomin1');
```

**方案2：完整迁移**
```javascript
// 使用新接口，数据更详细
const response = await fetch('/api/v1/strategy/v2/topics/recommend?ip_id=xiaomin1');
const { topics, match_stats } = await response.json();

// 新字段：
// - match_score: IP匹配度（0-1）
// - content_type: 内容类型（money/emotion/skill/life）
// - four_dim_score: 四维评分明细
// - match_stats: 统计信息
```

## 四、配置其他IP

### 添加新IP的内置选题库

编辑 `backend/app/services/builtin_topic_repository.py`，添加新IP的选题：

```python
NEW_IP_BUILTIN_TOPICS = {
    "money": [
        {"title": "xxx", "tags": ["xxx"], "score": 4.9},
        # ...
    ],
    "emotion": [...],
    "skill": [...],
    "life": [...],
}
```

然后在 `get_builtin_topics` 函数中添加分支：

```python
def get_builtin_topics(ip_id: str, ...):
    if ip_id == "xiaomin1":
        topics_db = XIAOMIN_BUILTIN_TOPICS
    elif ip_id == "new_ip_id":
        topics_db = NEW_IP_BUILTIN_TOPICS
    else:
        topics_db = GENERIC_BUILTIN_TOPICS
    # ...
```

## 五、监控与调优

### 关键指标监控

```python
# 在日志中关注以下指标
- 推荐返回率（应>95%）
- 平均匹配分数（应>0.7）
- 各内容类型分布（应接近4:3:2:1）
- TIKHUB成功率（作为数据源质量参考）
```

### 调优建议

| 问题 | 调优方案 |
|------|---------|
| 匹配分数偏低 | 补充IP画像信息；添加更多同义词 |
| 某类内容过多/过少 | 调整内容矩阵权重 |
| 选题重复 | 增加内置库选题数量；启用TIKHUB |
| 语义匹配慢 | 使用缓存；降级到关键词匹配 |

## 六、常见问题

### Q: 不安装sentence-transformers可以吗？
A: 可以，系统会自动回退到关键词匹配。建议安装以获得更好的匹配效果。

### Q: TIKHUB还需要吗？
A: 不强制需要，但建议保留。TIKHUB提供实时热榜数据，内置库提供稳定兜底。

### Q: 如何验证新系统工作正常？
A: 访问以下端点：
- `/api/v1/strategy/v2/topics/builtin` - 应返回12个选题
- `/api/v1/strategy/v2/topics/recommend` - 应返回12个选题（无论TIKHUB状态）
- `/api/v1/strategy/v2/topics/match-test` - 测试匹配算法

### Q: 内置选题会重复吗？
A: 每次返回会随机打乱顺序，且内置库选题数量充足（50+），短期不会明显重复。

## 七、回滚方案

如需回退到V1.0：
1. 移除新路由注册（main.py）
2. 前端切回旧接口 `/api/v1/creator/topics/recommended`

V1.0和V2.0可以共存，不影响原有功能。
